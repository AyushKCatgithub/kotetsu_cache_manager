# Contributing to kotetsu_cache_manager

Thank you for your interest in contributing to **kotetsu_cache_manager**.  
All contributions are welcome, including improvements, bug fixes, new features, tests, and documentation.

## How to Contribute

### 1. Fork the repository
Click the **Fork** button at the top of the GitHub page.

### 2. Clone your fork
```bash
git clone https://github.com/AyushKCatgithub/kotetsu_cache_manager.git
cd kotetsu_cache_manager
```

### 3. Install dependencies
```bash
flutter pub get
```

### 4. Create a new branch
```bash
git checkout -b feature/your-feature-name
```

### 5. Make your changes
Keep code clean and consistent. Avoid unnecessary comments.

### 6. Run tests
```bash
flutter test
```

### 7. Commit your changes
```bash
git add .
git commit -m "Add: your message"
```

### 8. Push to your fork
```bash
git push origin feature/your-feature-name
```

### 9. Create a Pull Request
Go to your fork on GitHub → **New Pull Request** → Submit.

---

## Contribution Guidelines

- Keep PRs focused on a single change.
- Ensure your code follows Dart & Flutter formatting:
  ```bash
  flutter format .
  ```
- Add tests when adding new features.
- Update the README or CHANGELOG when applicable.
- Make sure all tests pass before submitting.

---

## Reporting Issues

If you find a bug or want a new feature:

1. Go to the GitHub **Issues** tab
2. Create a new issue describing:
    - The problem
    - Expected behavior
    - Steps to reproduce
    - Any helpful snippets
